## 1. Resumen Ejecutivo

**GameWorld** es una aplicación web desarrollada como Trabajo de Fin de Máster cuyo objetivo principal es el diseño e implementación de una plataforma digital interactiva orientada al sector gaming.

El proyecto consiste en el desarrollo de una aplicación web tipo e-commerce que integra:

* Sistema de navegación modular
* Gestión de usuarios (registro y autenticación)
* Simulación de flujo de compra
* Panel de administración
* Integración de servicios backend (Firebase)
* Despliegue en entorno real

El desarrollo del proyecto ha permitido aplicar conocimientos adquiridos durante el máster en áreas como arquitectura web, desarrollo frontend, integración de servicios en la nube y despliegue en producción.

---

## 2. Justificación y Contexto

El sector gaming representa uno de los mercados digitales con mayor crecimiento a nivel global. La creación de una plataforma temática permite simular un entorno realista de comercio electrónico enfocado a este nicho, aplicando principios de:

* Desarrollo web moderno
* Diseño centrado en el usuario (UX/UI)
* Modularización del código
* Integración de servicios externos
* Escalabilidad estructural

Este proyecto actúa como demostración práctica de competencias técnicas en el desarrollo completo de una aplicación web funcional.

---

## 3. Objetivos del Proyecto

### Objetivo General

Diseñar e implementar una aplicación web funcional que simule una plataforma e-commerce orientada al público gamer, aplicando buenas prácticas de desarrollo, estructuración y despliegue.

### Objetivos Específicos

* Diseñar una arquitectura web clara y modular.
* Implementar una interfaz responsive y accesible.
* Desarrollar un sistema de autenticación de usuarios.
* Simular la gestión de productos y carrito de compra.
* Integrar servicios backend mediante Firebase.
* Desplegar la aplicación en un entorno productivo.
* Garantizar mantenibilidad y escalabilidad básica del proyecto.

---

## 4. Arquitectura y Estructura Técnica

La aplicación sigue una arquitectura basada en frontend desacoplado con integración de servicios backend en la nube.

### 4.1 Estructura del Proyecto

```
gameworld-carlos/
│
├── index.html
├── catalogo.html
├── login.html
├── register.html
├── perfil.html
├── carrito.html
├── admin.html
├── sobre.html
│
├── css/                → Hojas de estilo
├── js/                 → Lógica de la aplicación
├── fotos/              → Recursos gráficos
├── audios/             → Recursos multimedia
│
├── package.json
└── firebase-debug.log
```

### 4.2 Organización del Código

* Separación de responsabilidades (estructura, estilo y lógica).
* Modularización por páginas funcionales.
* Gestión de eventos y manipulación del DOM mediante JavaScript.
* Integración de Firebase para autenticación y almacenamiento.
